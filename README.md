# ⚖️ NyaySaarthi

**NyaySaarthi** is an AI-powered legal assistant designed to simplify and accelerate legal workflows. It helps legal professionals manage cases efficiently through intelligent document analysis, automated summarization, legal query resolution, and predictive insights.

---

## 🚀 Features

- 📂 **Document Upload** – Upload legal documents and case details  
- 🤖 **AI & NLP Analysis** – Extract insights using advanced ML/NLP models  
- 📝 **Summarization** – Generate concise summaries of lengthy legal texts  
- 💬 **Legal Chatbot** – Get instant answers to legal queries  
- 📊 **Case Prediction** – Predict outcomes and suggest legal arguments  
- 🔗 **Law Mapping** – Map IPC–BNS, CrPC–BNSS, and IEA–BSA sections  
- 🌐 **Multilingual Support** – Access outputs in multiple languages  
- 📊 **Dashboard** – Centralized interface for all modules  

---

## 🏗️ System Architecture

### 🔹 Core Components

| Component | Description |
|----------|-------------|
| RAG | Retrieves legal data and generates accurate responses |
| LLM | Generates human-like legal text and insights |
| ML/NLP Models | Perform text analysis, classification, and prediction |
| Legal Corpus | Collection of legal documents and case laws |
| Insights Engine | Outputs summaries and predictions for decision-making |
| MongoDB | Stores legal data and user information |

---

### 🔹 Application Modules

| Module | Description |
|--------|-------------|
| Frontend (React Dashboard) | UI for uploading documents and viewing results |
| Backend (FastAPI) | Handles API requests and integrates components |
| Summarization | Converts long legal documents into concise summaries |
| Chatbot | Provides instant legal query responses |
| Prediction | Predicts case outcomes using historical data |
| Data Preprocessing | Cleans and prepares data for analysis |

---

## 🛠️ Tech Stack

- **Frontend:** React.js  
- **Backend:** FastAPI  
- **Database:** MongoDB  
- **AI/ML:** NLP, LLM, RAG Architecture  
- **Languages:** Python, JavaScript  

---

## ⚙️ How It Works

1. User uploads legal documents  
2. Data is preprocessed and cleaned  
3. NLP/ML models analyze the content  
4. RAG retrieves relevant legal information  
5. LLM generates summaries, insights, and responses  
6. Results are displayed via the dashboard
